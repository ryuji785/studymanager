# PRD - 学習計画プロトタイプ（LocalStorage）

## 目的とマーケ方針
- ターゲット: 資格勉強の社会人・高校生など「学習目標＋固定予定がある人」
- 主価値:
  1. 今週の学習可能時間を数値で可視化（画面の主役）
  2. 30分単位で現実的に週計画を作れる
  3. 完了チェックで「今週◯時間勉強した」が分かり安心できる（カテゴリ別内訳）

## 前提
- 自己管理（B2C）に統合。講師/生徒の分離は行わない。
- 週計画はスナップショットとして保存し、ユーザー操作でのみ再生成する。
- 永続化は LocalStorage（schemaVersion + migrate）。

## 主要機能
- 週計画（30分単位）
  - 週は月曜始まり
  - 学習ブロックは「カテゴリ＋時間」が必須、教材は任意
  - 完了ステータスで実績を集計
- 学習可能時間の可視化
  - 生活時間（睡眠・固定予定）を引いた週の残り時間
- 教材管理
  - 教材登録時にカテゴリ必須
  - カテゴリがなければその場で追加可能
- 自動生成
  - 教材期限から週タスク提案（便利機能）
  - 再生成は「追加/置換」選択、手動ブロックは維持

## 画面構成
- 週計画（主画面）
- 学習履歴（期間選択＋週一覧）
- 教材（カテゴリ管理含む）
- 設定（生活時間）

## フロー
- 初回: 生活時間のセットアップ → 週計画作成 → 学習ブロック追加
- 週次: 週切替 → 追加/編集 → 完了チェック → 実績確認
- 日次: タイムテーブル確認 → 完了を付ける

## 料金設計（将来）

- Pro: 自動生成・詳細レポート
